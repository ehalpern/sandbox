Config, Guice, Akka and Spray

Presents a light framework for building end-to-end async REST services in a simple, modular, testable fashion using Typesafe Config, Guice, Akka and Spray. The goal of this tutorial is to show a particular way to use these libraries together to build a an extensible application.

Configuration and Dependency Injection

This framework employs Guice dependency injection to modularize the system for clean extension and isolated testing. Guice is an excellent, lightweight implementation of the Java dependency injection standard JSR330.

The bindings of implementations (and config values) to be injected are specified in code using Module's (which we'll explore later).

You can make a service class injectable simply declaring all its dependencies as constructor arguments, and preceding the constructor argument list with the @Inject() annotation. The ClimateApi class uses this technique so that Guice can construct it with the appropriate implementation of ClimateService.

You can also include configuration properties to be injected in the constructor argument list. To do this, just annotate the argument with @Named("config-key") so that Guice knows what property to inject. The WbClimateClient uses this technique to inject the wbclimate.api.endpoint.

Dependency injection also helps simplify managing key implicit parameters. We bind the Akka ActorSystem, ActorRefFactory and the ExecutionContext so these can simply be injected into any class that requires them rather than propagating these arguments through the call stack. The ClimateApi class uses this technique to obtain the ExecutionContext. The WbClimateClient class uses this technique to inject the ActorRefFactory.
